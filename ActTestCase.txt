HOW MANY MODULES TO TEST
------------------------
You should plan **8 unit-level modules** for this Laravel BFP Inventory System.
Each module below gets its own Test Case ID (MT_UT_001 … MT_UT_008). Add more IDs
only if you split a module (e.g. separate “Export” from “Notifications”) or add
new features.

MODULE SUMMARY TABLE (fill names/dates when you execute)
--------------------------------------------------------------------------------
| # | Module / unit under test              | Test Case ID | Suggested priority |
|---|----------------------------------------|--------------|---------------------|
| 1 | Authentication (login, logout, guest)  | MT_UT_001    | High                |
| 2 | User management (CRUD users)           | MT_UT_002    | High                |
| 3 | Station management (CRUD stations)     | MT_UT_003    | High                |
| 4 | Item / inventory (per station)         | MT_UT_004    | High                |
| 5 | Transfer / movement between stations   | MT_UT_005    | High                |
| 6 | Dashboard (aggregates / landing)       | MT_UT_006    | Medium              |
| 7 | Reports                                | MT_UT_007    | Medium              |
| 8 | Notifications, activity, export        | MT_UT_008    | Medium              |

Pass / Fail / Not executed / Suspended (apply on every step and overall):
  Pass     — Expected Results match Actual Results.
  Failed   — Expected Results do not match Actual Results.
  Not executed / Suspended — Prerequisites missing, offline, data unavailable,
                            or blocked by another unit/module.

NOTES FOR YOUR SUBMISSION
================================================================================
• “Unit” in manual test documents often means one **cohesive module** (auth,
  users, items, …). Automated PHPUnit **Unit** tests would target classes
  (models, actions) in isolation; this file is structured for **manual** steps.
• Replace bracketed placeholders ([Name of Creator], [MM/DD/YY], Actual Results)
  when you run tests.
• If your instructor requires **one Word/Excel row per test case**, use the
  summary table for the cover sheet and attach one scenario table per MT_UT_*.
